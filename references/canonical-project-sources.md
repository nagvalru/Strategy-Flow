# Existing Project Sources

Use this reference when Strategy Flow runs inside a workspace that already contains domain-specific rules and templates.

## Purpose

Strategy Flow is a workflow layer. It should not dilute or replace stronger project-specific strategy rules that already exist in the workspace.

## Typical Existing Sources

Examples of project-specific sources that may already exist:

- a strategy implementation guide for the visual editor;
- a final strategy description template;
- a final indicator description template;
- a risk-management document;
- an indicator registry;
- an indicator-authoring guide.

## How Strategy Flow Should Behave

- Use Strategy Flow for phase control and gates.
- Use the existing project files for detailed domain rules, templates, and documentation standards.
- Do not create a second competing standard for strategy descriptions or indicator descriptions when these project templates already exist.
- If a new skill introduces a new step, route that step through the existing project rules instead of bypassing them.

## Documentation Rule

If the project already requires:

- strategy folder documentation;
- final strategy description;
- final indicator description;
- indicator registry updates;

then those outputs remain mandatory even if Strategy Flow adds new planning or verification phases.
